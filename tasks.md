## 결정 사항

| 결정 | 이유 |
| --- | --- |
| `tasks.md`, `findings.md`, `progress.md`의 3파일 구조를 유지한다 | 두 기준 문서 모두 계획, 발견, 진행 기록의 분리를 핵심으로 삼고 있다 |
| 계획 파일은 프로젝트 루트에서 관리한다 | 작업 시작 전 빠르게 읽고 갱신하기 쉽다 |
| 모든 마크다운 파일은 한글로 작성한다 | 프로젝트 운영 규칙과 사용자 요구를 일관되게 지키기 위함이다 |
| 작업 내용은 별도 지시가 없어도 주기적으로 계획 파일에 반영한다 | 파일 기반 계획 워크플로를 실제 작업 흐름과 분리하지 않기 위함이다 |
| 대화 히스토리는 `CHATS.md`에 누적 기록한다 | 세션이 바뀌어도 질문 흐름과 이전 맥락을 이어가기 위함이다 |
| `CHATS.md`는 매번 읽지 않고 사용자가 요청할 때만 확인한다 | 불필요한 반복 확인 없이 기록 저장 용도를 중심으로 운영하기 위함이다 |
| 현재 프로젝트 주제는 `ChatGPT로 유튜브 동영상 쉽게 제작하기`로 유지한다 | 이후의 계획과 산출물을 하나의 방향으로 정렬하기 위함이다 |
| 워크플로 기준 문서는 `planning-with-files.md`와 `file-based-planning-workflow.md` 두 파일을 함께 참고한다 | 이번 프로젝트의 로컬 기준 문서가 두 개로 확정되었기 때문이다 |
| 유튜브 제작의 사용자용 단순 흐름은 `주제 선정 -> 대본 작성 -> 영상 제작`으로 이해한다 | 전체 개념을 빠르게 설명하기 쉬운 최소 구조이기 때문이다 |
| 실제 작업 기준의 최종 흐름은 6단계 버전으로 운영한다 | 품질과 실행 가능성을 높이기 위해 중간 검증 단계를 포함해야 한다 |
| 최종 6단계 흐름은 `주제 선정 -> 주제 검증 및 영상 방향 정리 -> 유튜브용 대본 작성 -> 대본 검수 및 수정 -> 장면 구성표 작성 -> 동영상 제작`으로 본다 | 현재 프로젝트 목적에 가장 맞는 실사용 프로세스라고 판단했다 |
| 첫 대상 사용자는 `초보 유튜버`로 본다 | 설명 방식과 제작 가이드를 쉽게 설계하기 위함이다 |
| 첫 실험 주제는 `30대라면 반드시 먹어야 하는 영양제 3가지`로 정한다 | 6단계 흐름을 실제 사례에 적용해 보기 위한 기준 주제가 필요하다 |
| 벤치마킹은 `대본 복사`보다 `구조 분석` 중심으로 진행한다 | 저작권과 차별화 문제를 줄이고 재사용 가능한 제작 방식을 만들기 위함이다 |

## 메모

- 새로운 단계로 넘어갈 때마다 현재 단계를 갱신한다.
- 작업 전에 `AGENTS.md`, `planning-with-files.md`, `file-based-planning-workflow.md`를 우선 확인한다.
- 계획 변경이나 결정 사항이 생기면 즉시 이 파일에 반영한다.
- 이 파일은 해야 할 일과 현재 단계 추적에 집중한다.
- 세션 간 문맥 유지가 필요한 대화는 `CHATS.md`에 함께 기록한다.
- `CHATS.md`는 사용자가 보라고 할 때만 확인하고, 평소에는 업데이트만 수행한다.
- 현재 기준에서 가장 먼저 해야 할 일은 `주제 선정 기준`, `대상 사용자`, `첫 산출물`을 정하는 것이다.
- 이제 다음 우선 작업은 `벤치마킹 영상 분석 기준`과 `대본 생성 입력 양식`을 만드는 것이다.
- `benchmark-template.md`를 기준으로 유사 영상 5~10개를 분석하면 다음 단계로 넘어갈 수 있다.
- 사용자는 유사 영상 5~10개의 `제목, 링크, 도입부, 핵심 전개, 마무리` 중 가능한 범위까지 가져오면 된다.
- 현재는 `001.md`, `002.md`, `003.md`를 기반으로 첫 대본 초안까지 작성한 상태다.
- 현재는 첫 대본 초안을 10분 내외 설명형 구조로 확장한 상태다.

# 에이전트 지침

## 우선순위

- 작업을 시작하거나 프로젝트 지침 관련 질문에 답하기 전에 항상 먼저 `AGENTS.md`를 읽습니다.
- 임시 판단과 이 파일의 지침이 충돌하면 이 파일의 지침을 우선합니다.
- `File-based Planning Workflow` 관련 기준이 필요하면 프로젝트 내부의 `planning-with-files.md`와 `file-based-planning-workflow.md`를 먼저 읽습니다.

## 언어

- 사용자 질문에 대한 모든 답변은 반드시 한글로 작성합니다.
- 마크다운 파일 작성은 반드시 한글로 합니다.

## 파일 기반 계획 워크플로

- 이 프로젝트는 `File-based Planning Workflow`를 기본 작업 방식으로 사용합니다.
- 워크플로 기준 문서는 프로젝트 내부의 `planning-with-files.md`와 `file-based-planning-workflow.md`를 함께 사용합니다.
- 파일 시스템을 계획과 실행을 위한 영구 작업 메모리로 취급합니다.
- 프로젝트 루트에서 다음 파일들을 유지하고 업데이트합니다.
  - `tasks.md`: 프로젝트 목표, 현재 단계, 체크리스트, 열린 질문, 결정 사항, 오류 기록
  - `findings.md`: 조사 메모, 기술적 발견, 구현 결정, 이슈, 학습 내용
  - `progress.md`: 세션별 작업 기록, 완료된 작업, 막힌 점, 다음 단계
  - `CHATS.md`: 사용자와의 대화 히스토리, 주요 질문, 답변 요약, 세션 간 문맥 연결 기록

## 워크플로 규칙

- 다른 프로젝트 지침 파일을 보기 전에 먼저 `AGENTS.md`를 읽습니다.
- 의미 있는 작업을 시작하기 전에 `tasks.md`, `findings.md`, `progress.md`를 읽습니다.
- `CHATS.md`는 사용자가 확인하라고 명시한 경우에만 읽습니다.
- 단계, 계획, 결정, 오류 기록이 바뀌면 `tasks.md`를 업데이트합니다.
- 중요한 기술 맥락을 발견하거나 의미 있는 결정을 내리면 즉시 `findings.md`를 업데이트합니다.
- `progress.md`는 기존 내용을 덮어쓰지 말고 세션마다 이어서 기록합니다.
- 사용자와의 대화 내용은 `CHATS.md`에 계속 누적 기록합니다.
- 사용자가 따로 지시하지 않아도 수행한 작업 내용은 `tasks.md`, `findings.md`, `progress.md`에 주기적으로 반영합니다.
- 작업 중 계획 변경, 발견사항 추가, 진행 상태 변화가 생기면 적절한 파일에 스스로 업데이트합니다.
- 세션이 바뀌어도 이전 질문 흐름을 이어갈 수 있도록 주요 질문과 답변 요약을 `CHATS.md`에 남깁니다.
- 다만 `CHATS.md`를 매번 확인하지는 않고, 사용자가 확인을 요청할 때만 읽습니다.

## 파일 구조 지침

- `AGENTS.md`, `tasks.md`, `findings.md`, `progress.md`, `CHATS.md`는 프로젝트 루트에 둡니다.
- Gemini 관련 실행 스크립트 파일은 `gemini` 폴더 아래에 둡니다.
- TTS 생성 스크립트 파일은 `tts_script` 폴더 아래에 둡니다.
- 영상 조립 스크립트 파일은 `video_script` 폴더 아래에 둡니다.
- 작성한 대본 파일은 `script` 폴더 아래에 둡니다.
- `tasks.md`는 계획용, `findings.md`는 지속적으로 참고할 발견사항용, `progress.md`는 시간순 작업 이력용으로 사용합니다.
- `CHATS.md`는 세션을 넘겨 이어봐야 하는 질문 맥락과 답변 이력을 기록하는 용도로 사용합니다.
- 평소에는 기록만 갱신하고, 내용 확인은 사용자가 요청한 경우에만 수행합니다.
- 명시적 요청이 없다면 계획, 기술 발견, 세션 로그를 하나의 파일에 섞어 쓰지 않습니다.

## 운영 원칙

- 프로젝트 맥락이 부족하면 먼저 계획 파일을 만들거나 갱신해서 이후 작업에 지속적인 문맥을 남깁니다.
- 큰 수정 전에, 새로운 단계로 넘어가기 전에, 또는 작업 방향이 불분명해질 때 이 지침 파일들을 다시 확인합니다.
- 작업 완료 보고 전에 관련 계획 파일들이 최신 상태인지 먼저 점검하고 필요하면 갱신합니다.
- 사용자와의 대화가 중요한 문맥을 포함하면 답변 후 `CHATS.md`를 최신 상태로 유지합니다.
- 새로 작성하는 대본은 루트가 아니라 반드시 `script` 폴더 하위에 생성합니다.
- Gemini 이미지 생성 스크립트는 `script`가 아니라 반드시 `gemini` 폴더 하위에 둡니다.
- TTS 생성 스크립트는 반드시 `tts_script` 폴더 하위에 둡니다.
- 영상 조립 스크립트는 반드시 `video_script` 폴더 하위에 둡니다.

## 이미지 생성 지침

- 무료 모델인 `나노바나나` 계열을 사용할 때는 이미지 생성 프롬프트에 한글 텍스트를 넣지 않습니다.
- 무료 모델 이미지에는 화면 속 문구, 포스터 문구, 자막 문구처럼 읽히는 한글 텍스트 요소를 포함하지 않습니다.
- 무료 모델 이미지 프롬프트는 가능하면 영어 중심으로 작성하고, 텍스트 없는 구성으로 설계합니다.

## 참고 대본 지침

- 샘플 대본과 벤치마킹용 참고 대본은 `refer` 폴더 아래에 둡니다.
- `refer` 폴더는 대본 작성 시 참고하는 샘플 대본 보관용으로 사용합니다.
- 사용자가 제공한 샘플 대본 파일은 가능하면 루트가 아니라 `refer` 폴더 하위에 정리합니다.
- 실제 작성 결과물은 `script` 폴더 하위에 두고, 참고 자료는 `refer` 폴더 하위에 둬서 서로 구분합니다.

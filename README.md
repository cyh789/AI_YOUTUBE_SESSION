1. 프로젝트 주제, 'ChatGPT로 유튜브 동영상 쉽게 제작하기'  
- 3/22~24일, 3일에 걸쳐 프로젝트 진행  

2. 전체 작업은 'file-based-planning-workflow'를 기준으로 진행  
- 'tasks.md'에는 단계, 체크리스트, 결정 사항 기록  
- 'findings.md'에는 기술 발견, 구현 판단, 구조 변경 이유 누적  
- 'progress.md'에는 날짜와 세션 기준으로 실제 작업 흐름 기록  
- 문서가 커질수록 역할별 파일과 폴더로 분리하는 방향 선택, 필요하면 MD 파일 추가 분할도 가능한 구조로 정리

3. 'AGENTS.md'에 지침을 모아 작업 우선순위와 폴더 구조 관리  
- 모든 작업 전에 'AGENTS.md'를 먼저 확인하는 방식을 기준으로 설정  
- 다른 세션에서도 흐름을 이어가기 위해 'CHATS.md'에 대화 요약 누적  

4. 지침 파일이 계속 커지는 문제를 줄이기 위해 스킬 기반 구조 도입  
- 전체 흐름은 '분석 -> 대본 -> 장면계획 -> 이미지 -> TTS텍스트 -> TTS오디오 -> 영상조립 -> 업로드점검'으로 정리  
  1) 분석: 'refer' 폴더의 샘플 대본 구조를 분석하는 단계. 'project-reference-analyzer'  
  2) 대본: 분석 결과를 바탕으로 한국어 유튜브 대본을 'script' 폴더에 작성하는 단계. 'project-script-writer'  
  3) 장면계획: 대본을 장면 단위로 쪼개고 이미지와 내레이션 흐름을 설계하는 단계. 'project-scene-planner'  
  4) 이미지: 대본과 장면계획을 바탕으로 Gemini를 이용해 장면 이미지를 'output'에 생성하는 단계. 'project-script-image-generator'  
  5) TTS텍스트: 이미지와 대본을 맞춰 장면별 한국어 TTS 텍스트를 'tts'에 만드는 단계. 'project-scene-tts-builder'  
  6) TTS오디오: 'tts_script'의 스크립트로 텍스트를 실제 음성 파일로 변환하는 단계  
  7) 영상조립: 이미지와 오디오를 묶어 최종 영상으로 합치는 단계. 'project-video-assembler'  
  8) 업로드점검: 업로드 전 제목, 설명, 썸네일, 자동 자막 검수 등을 확인하는 단계. 'project-upload-checker'

5. 실제 작업은 역할별 폴더를 분리하여 진행  
- 'refer'는 샘플 대본, 'script'는 작성 대본, 'gemini'는 이미지 생성 스크립트, 'output'은 생성 이미지, 'tts'는 TTS 텍스트, 'tts_script'는 음성 생성 스크립트, 'video_script'는 영상 조립 스크립트로 구분  
- 즉, AI를 한 번 쓰고 끝내는 방식이 아니라 폴더 구조와 문서 구조까지 포함한 제작 파이프라인으로 정리

6. 세션이 달라져도 이어서 작업할 수 있도록 기록 중심으로 운영  
- 'CHATS.md'에는 대화 흐름 기록  
- 'tasks.md', 'findings.md', 'progress.md'에는 계획, 발견, 진행 상황을 분리 기록  
- 결과적으로 이번 프로젝트는 AI를 단순 생성 도구로 사용한 것이 아니라, 문서화된 지침, 스킬, 스크립트, 폴더 구조를 함께 쌓아가며 반복 가능한 제작 시스템으로 발전시키는 방식으로 진행

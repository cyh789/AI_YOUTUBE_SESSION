# Project Scene TTS Builder

이 스킬은 장면 이미지와 대본을 바탕으로 TTS용 텍스트 파일을 만들 때 사용합니다.

## 역할

- `output` 폴더의 생성 이미지를 읽습니다.
- `script` 폴더 하위의 대본을 읽습니다.
- 필요하면 `scene-plan-01.md` 같은 장면 구성표를 함께 참고합니다.
- 장면별로 읽기 쉬운 한국어 TTS 텍스트를 만듭니다.
- 결과를 `tts` 폴더 하위 텍스트 파일로 저장합니다.

## 핵심 규칙

- 입력 이미지는 `output` 폴더 하위에서 찾습니다.
- 입력 대본은 `script` 폴더 하위에서 찾습니다.
- 결과물은 이미지가 아니라 한국어 TTS 텍스트 파일입니다.
- 결과물은 `tts` 폴더 하위에 저장합니다.
- 목적은 10~20분 유튜브 영상 조립을 위한 장면별 내레이션 텍스트를 만드는 것입니다.
- 텍스트는 현재 주제와 현재 장면 이미지에 맞아야 합니다.
- 문장은 TTS에 바로 넣을 수 있도록 짧고 자연스러운 구어체로 작성합니다.

## 작업 순서

1. `AGENTS.md`를 읽습니다.
2. `tasks.md`, `findings.md`, `progress.md`를 읽고 현재 프로젝트 상태를 파악합니다.
3. `script` 폴더 하위의 대상 대본을 읽습니다.
4. `output` 폴더 하위의 대상 이미지를 확인합니다.
5. 필요하면 `scene-plan-01.md`를 읽고 장면 순서와 역할을 확인합니다.
6. 각 장면에 맞는 한국어 TTS 문장을 1~2문장 단위로 작성합니다.
7. 결과를 `tts/scene-xx-*.txt` 형식으로 저장합니다.
8. 작업 결과를 `tasks.md`, `findings.md`, `progress.md`, `CHATS.md`에 반영합니다.

## 작성 기준

- 너무 긴 문장보다 끊어 읽기 쉬운 짧은 문장을 우선합니다.
- 이미지 분위기와 대본 내용이 어긋나지 않도록 맞춥니다.
- 초보 유튜버가 바로 TTS 엔진에 넣어 쓸 수 있는 형태를 목표로 합니다.
- 건강 정보 주제는 과장 표현보다 설명형, 정보형 톤을 우선합니다.
- 장면 하나당 한 파일 원칙을 기본으로 하되, 필요하면 `a`, `b`처럼 나눌 수 있습니다.

## 결과물

- 기본 결과물: `tts` 폴더 하위의 장면별 한국어 TTS 텍스트 파일
- 필요 시 부가 결과물: TTS 분할 메모, 장면별 읽기 톤 메모

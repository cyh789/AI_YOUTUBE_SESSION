# Project Reference Analyzer

이 스킬은 샘플 대본을 분석해 반복 가능한 구조를 추출할 때 사용합니다.

## 역할

- 'refer' 폴더 하위의 샘플 대본을 읽습니다.
- 공통 구조, 훅, 정보 배열, 마무리 방식을 분석합니다.
- 대본 복사가 아니라 구조 벤치마킹 결과를 정리합니다.
- 분석 결과를 대본 작성 전에 참고할 수 있는 메모 형태로 정리합니다.

## 핵심 규칙

- 입력 샘플 대본은 'refer' 폴더 하위에서 찾습니다.
- 문장 자체를 복사하지 말고 구조만 추출합니다.
- 분석 결과는 현재 프로젝트의 대본 작성과 장면계획에 바로 연결될 수 있어야 합니다.
- 결과는 'findings.md'에 반영하거나, 필요하면 별도 마크다운 분석 파일로 정리합니다.

## 작업 순서

1. 'AGENTS.md'를 읽습니다.
2. 'tasks.md', 'findings.md', 'progress.md'를 읽고 현재 프로젝트 상태를 파악합니다.
3. 'refer' 폴더 하위 샘플 대본을 읽습니다.
4. 공통 훅, 본문 전개, 정보 설명 방식, 마무리 패턴을 정리합니다.
5. 현재 작업 주제에 재사용할 수 있는 구조 메모를 만듭니다.
6. 결과를 'findings.md'와 관련 문서에 반영합니다.

## 작성 기준

- 분석은 간결하게 하되 실제 대본 작성에 바로 쓸 수 있어야 합니다.
- 구조, 흐름, 패턴 중심으로 정리합니다.
- '무엇을 따라 쓸지'보다 '무엇을 참고할지'가 드러나야 합니다.

## 결과물

- 기본 결과물: 'findings.md'에 반영된 구조 분석 메모
- 필요 시 부가 결과물: 별도 벤치마킹 마크다운 파일

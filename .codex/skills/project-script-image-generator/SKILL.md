# Project Script Image Generator

이 스킬은 이 프로젝트의 대본을 바탕으로 장면용 이미지를 생성할 때 사용합니다.

## 역할

- `script` 폴더 하위의 대본을 읽습니다.
- 필요하면 `scene-plan-01.md` 같은 장면 구성표를 함께 참고합니다.
- 대본과 주제에 맞는 장면별 이미지 프롬프트를 만듭니다.
- Gemini 이미지 생성 스크립트를 호출해 PNG 이미지를 생성합니다.
- 결과 이미지를 `output` 폴더 하위에 저장합니다.

## 핵심 규칙

- 입력 대본은 `script` 폴더 하위에서 찾습니다.
- 결과 이미지는 `output` 폴더 하위에 PNG로 저장합니다.
- 이미지 생성 스크립트는 `gemini/gemini-image-generate.mjs`를 사용합니다.
- Gemini API 키는 `.env`의 `GEMINI_API_KEY`를 기본으로 사용하며, 필요하면 `GOOGLE_API_KEY`도 허용합니다.
- 무료 모델을 사용할 때는 이미지 안에 한글 텍스트를 넣지 않습니다.
- 무료 모델 프롬프트는 영어 중심으로 작성하고, `no text` 조건을 우선 적용합니다.
- 이미지의 목적은 한국어 유튜브 비디오 대본을 10~20분 영상으로 구성하기 위한 장면 이미지입니다.
- 샘플 이미지를 만드는 것이 아니라, 현재 주제와 현재 대본에 맞는 이미지를 생성해야 합니다.

## 작업 순서

1. `AGENTS.md`를 읽습니다.
2. `tasks.md`, `findings.md`, `progress.md`를 읽고 현재 프로젝트 상태를 파악합니다.
3. `script` 폴더 하위의 대상 대본을 읽습니다.
4. 필요하면 `scene-plan-01.md`를 읽고 장면 단위를 더 구체화합니다.
5. 각 장면에 맞는 이미지 프롬프트를 만듭니다.
6. 무료 모델이면 영어 프롬프트와 `no text` 구성을 적용합니다.
7. `gemini/gemini-image-generate.mjs`를 호출해 PNG 이미지를 생성합니다.
8. 결과를 `output/<파일이름>.png`로 저장합니다.
9. 작업 결과를 `tasks.md`, `findings.md`, `progress.md`, `CHATS.md`에 반영합니다.

## 프롬프트 작성 기준

- 장면의 목적이 한눈에 보이도록 작성합니다.
- 이미지 안에 자막이나 제목이 들어가야 할 것처럼 유도하지 않습니다.
- 건강 정보 주제는 신뢰감 있고 과장되지 않은 톤을 우선합니다.
- 초보 유튜버가 바로 영상에 넣기 쉬운 구성을 목표로 합니다.
- 같은 장면도 필요하면 2~3개의 변형 프롬프트로 나눠 시도할 수 있습니다.

## 결과물

- 기본 결과물: `output` 폴더 하위의 PNG 이미지 파일
- 필요 시 부가 결과물: 장면별 프롬프트 메모, 이미지 선택 메모
